Game-asset generation, running entirely on my own GPU.

A local-first pipeline that produces three kinds of game asset — images, music and pixel art — on a single AMD Radeon RX 9070 XT. ComfyUI node-graphs drive SDXL with upscale and detail passes; Stable Audio Open composes short pieces straight to MP3; and a custom Python server wires Stable Diffusion directly into Aseprite for LoRA-styled sprites. The twist: almost none of this officially targets AMD — so getting it onto a Radeon was half the project.

The pipeline

One graph, many passes

Image generation isn't a single model call — it's a ComfyUI node graph. SDXL lays down a base image, an upscaler raises resolution, and detail/segmentation passes clean up faces and fine structure before the refiner finishes the frame. Every stage is a node, wired by hand and reusable.

NVIDIA-targeted, AMD-powered

Each tool assumes a CUDA-capable NVIDIA card. The pixel-art server gets around that with a launcher that detects the GPU at startup and installs the right PyTorch backend — DirectML for the Radeon on Windows — so the same code runs on AMD with no source changes.

05

Notes from the build

i

One launcher, four backends

Rather than maintain separate setups, the startup script detects the GPU at launch and installs the matching PyTorch build — CUDA, DirectML, ROCm or CPU — then verifies the device is actually live before starting the server.

ii

DirectML as the AMD route

On Windows the trick is to install the CPU PyTorch wheel and add the torch-directml plugin, which exposes the Radeon as a torch device. No CUDA, no vendor lock-in — the same diffusers code just runs.

iii

Generation inside the editor

Pixel art is generated where it's used: the Aseprite plugin talks to a local Flask server, so a prompt becomes a sprite on the canvas without exporting, uploading or switching apps.

iv

Style, pinned by LoRA

Raw SDXL drifts in style. Two pixel-art LoRAs applied through peft keep characters and tiles coherent, so a set of assets actually looks like it belongs in the same game.

v

Local, private, and free to iterate

Everything runs on one machine. With offline mode the pipeline needs no network once weights are cached, nothing leaves the GPU, and there's no per-image bill — so experimenting is effectively free.
